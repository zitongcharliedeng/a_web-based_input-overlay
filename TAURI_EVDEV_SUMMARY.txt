ARCHITECTURE
================================================================================

Linux Kernel (/dev/input/event* devices)
    ↓
evdev Crate (rust-evdev = 0.12)
    ↓
Tauri Rust Backend (src/main.rs)
    ↓ IPC (window.emit())
JavaScript Frontend (browserInputOverlayView/)
    ↓
User Sees Overlay

KEY DESIGN DECISIONS:

✓ evdev over rdev:
  - Direct kernel API (no abstraction layer)
  - Works globally even when window unfocused
  - Guaranteed Wayland support
  - Used by OBS, Steam Input, AntiMicroX

✓ Tauri over Electron:
  - Smaller binary size (~50MB vs 150MB)
  - Better Linux integration
  - Native transparency support
  - Simpler build process

✓ All code inline (main.rs):
  - 429 lines total (no dependencies on external modules)
  - 100+ keyboard key mappings
  - Mouse tracking (position, buttons, wheel)
  - Gamepad analog stick support

================================================================================
PERMISSIONS REQUIREMENT
================================================================================

Why "sudo usermod -aG input $USER"?

Linux restricts access to /dev/input/ devices for security. The "input" group
is the standard permission model used by ALL professional input tools:

✓ OBS Studio
✓ Steam Input
✓ AntiMicroX (gamepad mapper)
✓ Key Mapper
✓ Input Leap (KVM software)
✓ Espanso (text expander)

This is NOT a security bypass - it's an intentional permission grant.
Users must opt-in to allow applications to read input devices.

After adding yourself to the input group, you must:
1. Log out and log back in, OR
2. Run: newgrp input

================================================================================
TROUBLESHOOTING
================================================================================

Problem: "No input devices found"
Solution: Check permissions
  - ls -l /dev/input/event0 (should show: crw-rw---- root input)
  - groups $USER (should include: input)
  - Log out and back in
  - sudo usermod -aG input $USER

Problem: "Permission denied" on /dev/input
Solution: Same as above - add to input group and re-login

Problem: Very high CPU usage
Solution: Reduce polling rate in main.rs
  - Change: thread::sleep(Duration::from_millis(1));
  - To: thread::sleep(Duration::from_millis(5));

Problem: Input not captured when unfocused
Solution: Check that global capture is actually NOT working (unlikely with evdev)
  - Verify overlay window is visible
  - Check /dev/input devices are readable
  - Check compositor is running (echo $WAYLAND_DISPLAY)

================================================================================
EVENT FORMAT (JavaScript Frontend)
================================================================================

Events are emitted as:

{
  type: "keydown" | "keyup" | "mousemove" | "mousedown" | "mouseup" | 
        "mousewheel" | "gamepadaxis" | "gamepadbutton" | "inputstatus",
  data: {
    // Event-specific fields
    key?: "W" | "SPACE" | "F1" etc.
    x?: number  // Mouse X
    y?: number  // Mouse Y
    button?: "left" | "right" | "middle"
    delta?: number  // Wheel amount
    direction?: "up" | "down" | "left" | "right"
    axis?: "leftStickX" | "rightTrigger" etc.
    value?: number  // Normalized -1.0 to 1.0
    pressed?: boolean  // For gamepad buttons
    timestamp: number  // Milliseconds since epoch
  }
}

Listen in JavaScript:
window.addEventListener('input-event', (event) => {
    console.log('Input:', event.detail);
});
