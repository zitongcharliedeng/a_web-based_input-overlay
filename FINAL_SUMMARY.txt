ARCHITECTURE HIGHLIGHTS
======================

Three-Layer Design:
  UI Components (React-like) 
    ↓
  AppState (Shared State / React Context API)
    ↓
  evdev Input Capture (Blocking Tokio task)

Component Model:
  - RSX macros (Rust version of JSX)
  - use_effect for lifecycle
  - use_shared_state for context
  - Pure functional components
  - Props via #[inline_props]

Input Capture:
  - /dev/input/event* direct access
  - 24-byte evdev event parsing
  - KEY, ABSOLUTE, RELATIVE event types
  - Blocking task (no async driver available)
  - Works on ALL Wayland compositors

DIOXUS HOOKS SYSTEM
===================

1. Shared State (React Context API)
   cx.provide_context(AppState::new());

2. Component Lifecycle (useEffect equivalent)
   use_effect(cx, (), |_| { /* init code */ });

3. Rendering (JSX equivalent)
   rsx!(cx, div { class: "app", h1 { "Title" } })

4. Background Tasks (async/spawn)
   cx.spawn(async { input::start_capture().await });

DIOXUS VS REACT COMPARISON
==========================

Similarities:
  ✓ Function components
  ✓ Hooks system (useState, useEffect)
  ✓ Context API
  ✓ Component composition
  ✓ Props passing
  ✓ Reactive rendering

Differences:
  Dioxus uses RSX (Rust macros) instead of JSX
  Dioxus has compile-time type safety
  Dioxus includes native renderer (not webview)
  Dioxus can access OS directly (evdev)
  React has instant hot reload, Dioxus recompiles

WHY RUST WINS FOR INPUT OVERLAYS
=================================

1. Direct Hardware Access
   Rust/evdev: Direct /dev/input access (pure Rust)
   JavaScript: NO access, must use native modules

2. Native Compilation
   Dioxus: 40-50MB binary, instant startup
   Electron: 200MB+ bundle, 2-3s startup (Chromium)

3. Type Safety
   Rust: Compiler enforces ALL event variants handled
   JS: Runtime errors, hard-to-debug type issues

4. Wayland Native
   evdev: Works on ALL Wayland compositors
   Electron: Experimental Wayland, needs workarounds

5. Performance
   Dioxus: 1-2ms input latency
   Electron: 5-10ms (JavaScript overhead)

BUILDING & RUNNING

  # Install: libxkbcommon-devel, libwayland-devel, pkg-config
  cargo build --release
  ./target/release/input-overlay
